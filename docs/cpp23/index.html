C++23 新特性概览

C++23 继续完善现代 C++，引入了 std::expected、deducing this、std::print、std::mdspan 等重要特性。部分特性编译器支持仍在进行中。

编译器支持

C++23 的部分特性在当前主流编译器（GCC 14、Clang 18、MSVC 2022）中支持不完整。本章包含 12 个可编译示例 和 7 个仅文档说明的特性。

Deducing this（显式对象参数）​

消除 const/非const 成员函数重载的代码重复。

传统方式 vs Deducing this
// C++23 之前需要写两个版本：
struct Widget {
    int& get() { return data; }
    const int& get() const { return data; }
    int data;
};

// C++23 用 deducing this 一个版本搞定：
struct Widget {
    // this auto&& self 是显式对象参数
    auto&& get(this auto&& self) {
        return std::forward_like<decltype(self)>(self.data);
    }
    int data;
};

关键点：

  • this auto&& self 作为第一个参数
  • 编译器根据调用者的 const 性自动推导 self 的类型
  • 也可用于 CRTP 简化
参考链接

deducing this

std::print / std::println​

替代 std::cout 的现代输出方式。

std::print 示例
#include <print>

int main() {
    std::println("Hello, {}!", "World");      // 自动换行
    std::print("pi = {:.4f}\n", 3.14159);     // 手动换行
    std::println("{:>10}", 42);                // 右对齐
}

关键点：

  • 基于 std::format 的格式化语法
  • println 自动追加换行符
  • 比 cout 更高效（减少了虚函数调用和 locale 处理）
  • 可以输出到任意输出流
参考链接

std::print | std::println

std::mdspan（多维数组视图）​

非拥有的多维数组视图，替代手动的多维索引计算。

std::mdspan 示例
#include <mdspan>

void process_matrix(std::mdspan<float, std::dextents<size_t, 2>> mat) {
    for (size_t i = 0; i < mat.extent(0); ++i)
        for (size_t j = 0; j < mat.extent(1); ++j)
            mat[i, j] *= 2.0f;
}

int main() {
    std::vector<float> data(12);
    // 将一维数据解释为 3x4 矩阵
    auto mat = std::mdspan(data.data(), 3, 4);
    process_matrix(mat);
}

关键点：

  • 支持任意维度
  • 支持多种布局策略（行优先、列优先、自定义步幅）
  • 与 std::span 类似但支持多维
  • 科学计算和图像处理的利器
参考链接

std::mdspan

std::flat_map / std::flat_set​

缓存友好的有序关联容器。

std::flat_map 示例
#include <flat_map>
#include <flat_set>

int main() {
    std::flat_map<std::string, int> scores;
    scores["Alice"] = 95;
    scores["Bob"] = 87;

    std::flat_set<int> s = {3, 1, 4, 1, 5};
    // 底层使用排序的 vector，迭代更快
}

关键点：

  • 底层使用排序的连续容器（通常是 vector）
  • 查找 O(log n)（二分搜索），但缓存更友好
  • 插入/删除 O(n)，适合读多写少的场景
  • 内存占用比 std::map（红黑树）更小
参考链接

std::flat_map | std::flat_set

std::generator​

标准的惰性生成器协程类型。

std::generator 示例
#include <generator>

std::generator<int> fibonacci() {
    int a = 0, b = 1;
    while (true) {
        co_yield a;
        auto next = a + b;
        a = b;
        b = next;
    }
}

int main() {
    for (auto n : fibonacci()) {
        if (n > 100) break;
        std::println("{}", n);
    }
}

关键点：

  • C++20 协程缺少标准的 Generator 类型，需要手写 promise_type
  • C++23 提供了 std::generator<T> 开箱即用
  • 支持 ranges 接口
  • 可以用于递归生成器（std::generator<T>::yielded）
参考链接

std::generator

static operator()​

无状态仿函数的零开销调用。

static operator() 示例
struct Hasher {
    static size_t operator()(int x) noexcept {
        return x * 2654435761u;
    }
};

// Lambda 也可以是 static
auto add = [](int a, int b) static { return a + b; };

关键点：

  • 无状态可调用对象不需要 this 指针
  • static operator() 可以被直接调用，无需对象实例
  • 编译器可以更好地优化

std::ranges 新增视图​

C++23 大幅扩展了 ranges 视图库。

C++23 新增视图示例
#include <ranges>

// chunk: 分块
auto chunks = v | std::views::chunk(3);
// [1,2,3], [4,5,6], [7,8,9]

// slide: 滑动窗口
auto windows = v | std::views::slide(3);
// [1,2,3], [2,3,4], [3,4,5], ...

// zip: 多个范围并行组合
auto zipped = std::views::zip(names, scores);

// enumerate: 带索引
for (auto [i, val] : v | std::views::enumerate) {
    std::println("[{}] = {}", i, val);
}

// chunk_by: 按条件分组
auto groups = v | std::views::chunk_by(std::less{});

// join_with: 用分隔符连接
auto joined = strs | std::views::join_with(',');

// cartesian_product: 笛卡尔积
for (auto [x, y] : std::views::cartesian_product(xs, ys)) {
    // ...
}

新增视图列表：

视图功能参考
chunk(n)按固定大小 n 分块chunk_view
slide(n)大小为 n 的滑动窗口slide_view
zip并行组合多个范围zip_view
zip_transform组合并转换zip_transform_view
enumerate带索引遍历enumerate_view
chunk_by(pred)按谓词分组chunk_by_view
join_with(sep)用分隔符连接join_with_view
stride(n)每隔 n 个取一个stride_view
cartesian_product笛卡尔积cartesian_product_view
as_rvalue转为右值视图as_rvalue_view
as_const转为 const 视图as_const_view
repeat(val)无限重复repeat_view
repeat(val, n)有限重复repeat_view
